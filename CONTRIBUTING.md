# Contributing

Contributions that improve correctness, security, interoperability, tests, or documentation are welcome.

1. Open an issue describing the change unless it is a small documentation fix.
2. Create a focused branch.
3. Add or update tests.
4. Run the repository's documented validation commands.
5. Open a pull request with security and compatibility notes.

Do not include credentials, customer information, production transaction data, or copied third-party code without a compatible license and attribution.
